В общем в любом языке есть функции, это такие штуки, которые принимают что-то и возвращают какой-то результат или не возвращают ничего.

В C++ функция определяется таким синтаксисом:

return_type function_name(args) {
  // code
}

return_type это какой-то тип или void если функция ничего не возвращает args это список аргументов функции

Например это принимает x и возвращает его модуль;

int abs(int x) {
  if (x < 0) return -x;
  else return x;
}

принимает x и возвращает true, если он отрицательный, иначе false;

bool is_negative(int x) {
  return x < 0;
}

принимает x и возвращает true, если это символ цифры и false иначе;

bool is_digit(char x) {
  return '0' <= x && x <= '9';
}

принимает a и b и возвращает true, если a меньше b и false иначе;

bool less(int a, int b) {
  return a < b;
}

заменяет все символы from в строке на символы to

void replace(std::string const& str, char from, char to) {
  for (int i = 0; i < str.size(); ++i) {
    if (str[i] == from) str[i] = to;
  }
}

После объявления ее можно вызвать, передав ей аргументы, например

int main() {
  std::cout << abs(-10) << "\n"          // 10
            << abs(10) << "\n"           // 10
            << is_negative(10) << "\n"   // false
            << is_negative(-10) << "\n"  // true 
            << less(1, 0) << "\n";       // false
}

Или можно было изначально написать например такой код, тут функции эквивалентны тем что выше, просто реализованы друг через друга.

bool less(int a, int b) {
  return a < b;
}

bool is_negative(int a) {
  return less(a, 0);
}

int abs(int x) {
  if (is_negative(x))
    return -x;
  else
    return x;
}

Так вот, это обычные функции, и в C++ они не могут быть вложены ни в какие другие объявления, то есть нельзя объявить функцию внутри другой функции.

Такой код не скомпилируется

void f() {  // объявление первой функции
  void g() { // объявление второй функции внутри первой
    // do something
  }
}
